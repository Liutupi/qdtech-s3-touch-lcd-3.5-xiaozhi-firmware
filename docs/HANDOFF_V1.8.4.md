# v1.8.4 交接补充（2026-07-23）

- 基线为已发布的 QDTech 3.5 寸 `v1.8.3`；此版本升级为 `v1.8.4`。
- “摇一摇实验室”新增全中文入口“掌卦”：三枚钱币动态旋转，六爻按自下而上的顺序落定，随后显示卦名、卦辞与现代解读。
- 六十四卦的图片、卦辞、解读与六爻数据均由 SD 卡 `/shake_lab/divination/` 读取。资源生成器为 `tools/generate_divination_sd_pack.py`；将生成的 `shake_lab` 目录复制至 SD 卡根目录。
- 已定位并修复摇动完成、结果即将显示时的重启：串口确认根因是 `bmi270` 采样任务栈溢出（`***ERROR*** A stack overflow in task bmi270`），而非 SD 图片错误。BMI270 任务现在只负责 I2C 采样，状态转换会投递到主应用线程，再获取 LVGL 锁执行。任务栈从 4096 增加到 6144。
- 掌卦结果图改为应用管理的 RGB565 画布缓冲，避免 LVGL 图片缓存重复释放。掌卦中文字统一使用中文字体，左侧浅色卦图区改为深色文字以保证可读。
- 构建已通过 ESP-IDF 5.5.2 `idf.py -B build-v181-fast-provisioning-off build`。仅写入 App 到 `0x100000` 与 `0x800000`，两个 OTA 槽均通过 esptool digest 校验；未写入 NVS、模型或分区表。
- 修复前日志已准确捕获并定位；修复后完成启动、Wi-Fi、MQTT、手机配置页和 BMI270 空闲采样确认。下一次实体验收优先执行完整“掌卦”摇动流程。
